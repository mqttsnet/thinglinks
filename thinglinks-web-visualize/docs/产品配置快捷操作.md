# 产品配置快捷操作

本文用于快速完成大屏产品配置和组件版本维护。

文中的命令可以从仓库内任意目录执行，包括大屏的 `docs` 目录。每个代码块都会先通过 Git 定位仓库根目录，再兼容独立仓库与 `thinglinks` 单仓库两种目录结构。代码块内不包含 shell 注释，可以直接使用代码块复制按钮。

每次只执行当前操作对应的代码块。版本更新代码块会提示输入实际版本号，不需要修改命令文本。

## 配置入口

大屏组件根目录 `.thinglinks-product.env` 是产品身份、大屏组件版本、发行信息、授权信息和公开运行标识的配置来源。

- 大屏版本以当前组件的 `.thinglinks-product.env` 为来源；同批发布时，使用 `product:set-version` 将其更新为同发行 Web 清单中的版本值。
- 修改其他产品配置时只编辑 `.thinglinks-product.env`，随后执行 `product:render`。
- 不分别手工维护 `package.json` 中由脚本管理的名称、版本和授权字段。
- 不在页面、语言包或构建代码中重复维护发行名称。
- 清单会纳入 Git，只保存可以进入前端构建产物的公开值，不保存密码、令牌或客户端密钥。

## 只读检查

```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
VISUALIZE_ROOT="$REPO_ROOT"
if [ -f "$REPO_ROOT/thinglinks-web-visualize/package.json" ]; then VISUALIZE_ROOT="$REPO_ROOT/thinglinks-web-visualize"; fi
cd "$VISUALIZE_ROOT"
pnpm product:check
```

## 更新大屏版本

```bash
(
  REPO_ROOT="$(git rev-parse --show-toplevel)"
  VISUALIZE_ROOT="$REPO_ROOT"
  if [ -f "$REPO_ROOT/thinglinks-web-visualize/package.json" ]; then VISUALIZE_ROOT="$REPO_ROOT/thinglinks-web-visualize"; fi
  printf '请输入与 Web 一致的大屏新版本号：'
  IFS= read -r VISUALIZE_VERSION
  if [ -z "$VISUALIZE_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  cd "$VISUALIZE_ROOT" || exit 1
  pnpm product:set-version "$VISUALIZE_VERSION"
)
```

版本写入命令会自动同步清单与 `package.json` 并完成一致性检查，不需要紧接着再执行 `product:render`。

## 手工修改清单后渲染

```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
VISUALIZE_ROOT="$REPO_ROOT"
if [ -f "$REPO_ROOT/thinglinks-web-visualize/package.json" ]; then VISUALIZE_ROOT="$REPO_ROOT/thinglinks-web-visualize"; fi
cd "$VISUALIZE_ROOT"
pnpm product:render
```

REPO_ROOT="$(git rev-parse --show-toplevel)"

## 完整验证

```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
VISUALIZE_ROOT="$REPO_ROOT"
if [ -f "$REPO_ROOT/thinglinks-web-visualize/package.json" ]; then VISUALIZE_ROOT="$REPO_ROOT/thinglinks-web-visualize"; fi
cd "$VISUALIZE_ROOT"
pnpm product:test
pnpm product:check
pnpm build
git diff --check -- .
git status --short -- .
```

开发服务不会代替发布检查。准备同步或提交前，以 `product:check`、`product:test` 和生产构建结果为准。

## 同步到其他发行仓库

同步代码时保留目标仓库自身的 `.thinglinks-product.env` 和清单中 `THINGLINKS_SYNC_PROTECTED_PATHS` 指定的文件。同步完成后，在目标仓库重新执行 `product:render`、`product:check`、`product:test` 和生产构建。

不要同步 `.git`、`node_modules`、`dist`、IDE 配置、工具缓存和运行时临时文件。

## 写锁提示

`product:render` 和 `product:set-version` 属于写操作，避免并发执行。出现产品配置写锁提示时，先等待正在运行的命令结束，不手工删除仍由活动进程持有的锁引用。

# 前端开发规范

## 目录结构规范

### 必须遵守的三级目录原则

此规范适用 不限于 components、api、routes、assets、images、views、locales 等等
为了快速定位功能所属应用及服务模块，请严格遵守此规范。

#### 应用名称

```properties
格式要求：全小写字母+数字（禁止特殊字符）
示例：
- iot（物联网业务系统平台统一应用名）
- card（物联网卡业务系统平台统一应用名）
```
#### 服务名称

```properties
格式要求：与后端微服务名称严格一致
典型值：
- system   ▸ 系统服务
- link     ▸ 物联服务
- rule     ▸ 规则引擎
```

#### 模块名称

```properties
格式要求：使用业务术语（禁用技术术语）
┌───────────────┬─────────────────────────────┐
| 业务场景       | 推荐命名                     |
├───────────────┼─────────────────────────────┤
| 设备OTA升级   | ota                         | 
| 产品管理       | product                     |
| 实时监控       | monitor                     |
└───────────────┴─────────────────────────────┘

```


### 强制规则

- 服务名称必须与后端 spring.application.name 中配置的服务名完全一致
- 模块目录内禁止出现common/util等通用性目录

### components 目录:应用隔离与通用层

`src/components/` 同样遵守应用维度隔离:

- **应用专属组件** → `src/components/{应用}/...`,与 `views`、`api` 的应用维度对齐。
  - 示例:`src/components/iot/`(物联网应用组件)、`src/components/video/`(视频应用组件);后续新应用(如 `ai`)同级新建 `src/components/ai/`。
  - 判定:该组件**只被某一个应用引用** → 归该应用目录。
- **全局通用组件** → 放 `src/components/` **外层**,不加应用前缀。
  - 指与具体业务应用无关、任意应用可复用的基础组件,如 `Table`、`Form`、`Modal`、`Icon`、`Page`。
  - 通用组件目录内**禁止**再包含任何应用专属业务组件。
- 组件独立子目录、`PascalCase` 命名、带 `index.ts` 导出;应用目录用顶层 `index.ts` 聚合。
- 目录名用**应用名**(`iot`),禁止用产品名(如 `Thinglinks/`)。

## 权限编码与目录映射规则

### 编码格式

```
module:section:feature:action
```

| 层级 | 含义 | 示例 |
|---|---|---|
| `module` | 业务模块 | `video`、`link` |
| `section` | 菜单组 | `device`、`media` |
| `feature` | 功能页 | `info`、`channel`、`server` |
| `action` | 操作 | `add`、`edit`、`delete` |

- 菜单 code = 3 段：`module:section:feature`
- 按钮 code = 父菜单 code + `:action`
- 隐藏页 code = 父菜单 code + `:detail`
- feature 使用**简短英文单词**，禁止使用 Java 实体类名

### 编码 → 文件路径推导

以 `video:device:info` 为例：

| 类型 | 路径 |
|---|---|
| 页面 | `src/views/video/device/info/` |
| API | `src/api/video/device/info.ts` |
| Model | `src/api/video/device/model/infoModel.ts` |
| 国际化 | `src/locales/lang/{locale}/video/device/info.ts` |
| DB component | `/video/device/info/index` |

### 页面目录结构

```
src/views/{module}/{section}/{feature}/
  ├── index.vue            # 列表页
  ├── detail.vue           # 详情页（可选）
  ├── Edit.vue             # 编辑弹窗（可选）
  └── {feature}.data.tsx   # 表格列、表单 schema
```

### 详情页 Tab 子功能

不注册独立路由，作为组件被 `detail.vue` 引入。目录独立存在，按钮权限挂在父详情页资源下。

从编码 `video:device:alarm:handle` 可推断文件在 `src/views/video/device/alarm/`。

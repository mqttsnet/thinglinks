# 产品配置快捷操作

本文用于快速完成 Cloud 产品配置、组件版本和 Util 依赖版本的维护。

文中的命令可以从仓库内任意目录执行，包括 `docs` 目录。每个代码块都会通过 Git 自动定位 Cloud 工程；既适用于独立 Cloud 仓库，也适用于 `thinglinks/thinglinks-cloud` 目录结构。代码块内不包含 shell 注释，可以直接使用代码块复制按钮。

每次只执行当前操作对应的代码块。版本更新代码块会提示输入实际版本号，不需要修改命令文本。

## 配置入口

Cloud 工程根目录的 `.thinglinks-product.env` 是产品身份、组件版本、发行信息、授权信息和消息队列命名空间的配置来源。脚本负责把配置同步到受管文件。

- 只修改版本时使用对应的版本命令。
- 修改其他产品配置时只编辑 `.thinglinks-product.env`，随后执行 `render`。
- 不分别手工修改 Maven 版本、过滤配置或消息队列命名空间。
- 清单会纳入 Git，不在其中保存密码、令牌或客户端密钥。

## 只读检查

检查清单、Maven 引用、运行标识和同步边界：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
"$CLOUD_ROOT/scripts/product-config.sh" check
```

查看当前 Cloud 组件版本：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
"$CLOUD_ROOT/scripts/product-config.sh" get-component-version
```

## 更新版本

更新 Cloud 组件版本，并同步受管文件：

```bash
(
  GIT_ROOT="$(git rev-parse --show-toplevel)"
  CLOUD_ROOT="$GIT_ROOT"
  if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
  printf '请输入 Cloud 新版本号：'
  IFS= read -r CLOUD_VERSION
  if [ -z "$CLOUD_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$CLOUD_ROOT/scripts/product-config.sh" set-component-version "$CLOUD_VERSION"
)
```

更新 ThingLinks Util 依赖版本，并同步受管文件：

```bash
(
  GIT_ROOT="$(git rev-parse --show-toplevel)"
  CLOUD_ROOT="$GIT_ROOT"
  if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
  printf '请输入 Util 依赖新版本号：'
  IFS= read -r UTIL_VERSION
  if [ -z "$UTIL_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$CLOUD_ROOT/scripts/product-config.sh" set-util-version "$UTIL_VERSION"
)
```

`scripts/bump-version.sh` 是 Cloud 组件版本命令的兼容快捷入口，自动化任务可以向它直接传入明确的版本号。

版本写入命令会自动完成渲染和一致性检查，不需要紧接着再执行 `render`。

## 手工修改清单后渲染

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
"$CLOUD_ROOT/scripts/product-config.sh" render
```

## 完整验证

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
"$CLOUD_ROOT/scripts/tests/product-config-test.sh"
"$CLOUD_ROOT/scripts/product-config.sh" check
mvn --batch-mode -f "$CLOUD_ROOT/pom.xml" -DskipTests clean package
git -C "$CLOUD_ROOT" diff --check -- .
git -C "$CLOUD_ROOT" status --short -- .
```

如果同时发布新的 Util 版本，先在 Util 仓库完成本地安装，再更新这里的 Util 依赖版本并构建 Cloud。

## 同步到其他发行仓库

同步代码时保留目标仓库自身的 `.thinglinks-product.env` 和清单中 `THINGLINKS_SYNC_PROTECTED_PATHS` 指定的文件。代码同步完成后，在目标仓库重新执行 `render`、`check`、配置回归和构建检查。

不要同步 `.git`、IDE 配置、构建产物、依赖缓存和运行时临时文件。

## 写锁提示

`render` 和版本更新属于写操作，避免并发执行。出现产品配置写锁提示时，先等待正在运行的命令结束，不手工删除仍由活动进程持有的锁引用。

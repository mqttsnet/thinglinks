# 数据库初始化指南

ThingLinks Cloud 使用一个 MySQL 默认库，并为每个租户创建独立的 MySQL 业务库和 TDengine 时序库。

## MySQL 数据库

| 数据库 | 用途 | 脚本说明 |
| --- | --- | --- |
| `thinglinks_ds_c_defaults` | 公共配置、权限、字典和租户元数据 | [`mysql/default/`](mysql/default/) |
| `thinglinks_base_1` | 内置租户 1 业务库 | [`mysql/tenant/`](mysql/tenant/) |
| `thinglinks_job` | 定时任务调度 | [`thinglinks-job/docs/db/mysql`](../../../thinglinks-job/docs/db/mysql/README.md) |

初始化 Cloud 默认库和内置租户库：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -d "$GIT_ROOT/thinglinks-cloud/docs/db/mysql" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
"$CLOUD_ROOT/docs/db/mysql/default/migration/verify-default-database.sh"
"$CLOUD_ROOT/docs/db/mysql/tenant/migration/verify-tenant-migration.sh"
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS thinglinks_ds_c_defaults CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;"
mysql -u root -p thinglinks_ds_c_defaults < "$CLOUD_ROOT/docs/db/mysql/default/baseline/thinglinks_ds_c_defaults.sql"
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS thinglinks_base_1 CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;"
mysql -u root -p thinglinks_base_1 < "$CLOUD_ROOT/docs/db/mysql/tenant/baseline/thinglinks_base_1.sql"
```

导入后按[默认库基线说明](mysql/default/baseline/README.md)设置管理员密码，并替换应用密钥、数据源密码、地图密钥、通知渠道和外部控制台地址等占位值。占位值未处理完之前不要启动 Cloud。

已有环境不要重新导入全量基线，应按 [MySQL 脚本说明](mysql/README.md)依次执行尚未执行的增量脚本。

## TDengine 数据库

TDengine 租户库命名为 `thinglinks_extend_{tenantId}`。初始化内置租户时执行：

```sql
CREATE DATABASE IF NOT EXISTS thinglinks_ds_c_defaults
  VGROUPS 10
  BUFFER 10
  PRECISION 'ns';

CREATE DATABASE IF NOT EXISTS thinglinks_extend_1
  VGROUPS 10
  BUFFER 10
  PRECISION 'ns';
```

| 参数 | 值 | 说明 |
| --- | --- | --- |
| `VGROUPS` | 10 | 虚拟节点组数量 |
| `BUFFER` | 10 | 每个 vnode 的写入缓存大小，单位 MB |
| `PRECISION` | ns | 时间戳精度 |

## 新增租户

平台新增租户时，由租户初始化流程创建 `thinglinks_base_{tenantId}` 和 `thinglinks_extend_{tenantId}`。后续数据库升级要对所有存量租户库逐库执行租户增量脚本，并记录每个库的执行和验证结果。

# 如何启动项目

## 导入工程

1. 将 `thinglinks-cloud` 导入 IDEA。
2. 将 `thinglinks-util` 导入 IDEA。
3. 先安装 Util，再编译 Cloud。

## 编译 ThingLinks Util

以下命令只安装到本机 Maven 仓库，并跳过 Javadoc、源码包和 GPG 签名：

```bash
mvn clean install -Dmaven.javadoc.skip=true -Dmaven.source.skip=true -Dgpg.skip=true -f pom.xml
```

## 编译 ThingLinks Cloud

在 Cloud 工程目录执行：

```bash
mvn clean package -DskipTests -f pom.xml
```

## IDEA 控制台编码

macOS 的 IDEA Maven Runner 可在 `Environment variables` 中设置：

```text
JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF-8
```

Windows 的 IDEA Maven Runner 可在 `VM Options` 中设置：

```text
-Dfile.encoding=UTF-8
```

## 统一升级版本号

Cloud 组件版本和 Cloud 使用的 Util 依赖版本由 Cloud 工程根目录产品清单维护。Util 自身采用独立发布周期。完整操作和任意目录执行方式见[产品配置快捷操作](产品配置快捷操作.md)。

更新 Cloud 组件版本：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
printf '请输入 Cloud 新版本号：'
IFS= read -r CLOUD_NEW_VERSION
"$CLOUD_ROOT/scripts/product-config.sh" set-component-version "$CLOUD_NEW_VERSION"
```

更新 Util 依赖版本：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
printf '请输入 Util 依赖新版本号：'
IFS= read -r UTIL_NEW_VERSION
"$CLOUD_ROOT/scripts/product-config.sh" set-util-version "$UTIL_NEW_VERSION"
```

Web 管理端 OAuth 客户端标识属于前端构建配置，由 Web 仓库根目录产品清单维护；Cloud 的运行时 OAuth 客户端注册由部署环境管理，不在源码清单中保存。

代码生成器的 Web 配置键已经规范为 `webVbenConfig`。使用旧配置的部署环境请按[产品配置迁移记录](../changelogs/product-config-migration.md)同步调整 YAML 键。

## 打标签

标签版本来源为根目录产品清单中的 `THINGLINKS_COMPONENT_VERSION`，对应当前构建版本：

```bash
GIT_ROOT="$(git rev-parse --show-toplevel)"
CLOUD_ROOT="$GIT_ROOT"
if [ -x "$GIT_ROOT/thinglinks-cloud/scripts/product-config.sh" ]; then CLOUD_ROOT="$GIT_ROOT/thinglinks-cloud"; fi
CLOUD_VERSION="$("$CLOUD_ROOT/scripts/product-config.sh" get-component-version)"
git -C "$GIT_ROOT" tag "$CLOUD_VERSION"
git -C "$GIT_ROOT" push origin "$CLOUD_VERSION"
```

# 产品配置快捷操作

本文用于快速完成 BifroMQ 插件产品配置、组件版本和兼容性版本维护。

文中的命令可以从社区仓库内任意目录执行，包括 `docs` 目录。每个代码块都会通过 Git 自动定位 BifroMQ 插件组件目录。代码块内不包含 shell 注释，可以直接使用代码块复制按钮。

每次只执行当前操作对应的代码块。版本更新代码块会提示输入实际值，不需要修改命令文本。

## 配置入口

BifroMQ 插件组件根目录 `.thinglinks-product.env` 是产品身份、插件组件版本、兼容性版本、发行信息、授权信息和 Maven 坐标的配置来源。

各版本字段含义不同，按实际变更分别更新：

| 维度 | 清单字段 | 用途 |
| --- | --- | --- |
| 插件组件版本 | `THINGLINKS_COMPONENT_VERSION` | 插件集合自身的发布版本 |
| BifroMQ 版本 | `THINGLINKS_BIFROMQ_VERSION` | 插件编译使用的 BifroMQ SPI 兼容版本 |
| Util 版本 | `THINGLINKS_UTIL_VERSION` | 认证插件依赖的 ThingLinks Util 版本 |
| Java 主版本 | `THINGLINKS_JAVA_VERSION` | 插件编译目标 Java 版本 |

- 版本变化使用对应的版本命令。
- 修改其他产品配置时只编辑 `.thinglinks-product.env`，随后执行 `render`。
- 不分别手工修改根 POM 或插件父 POM 中的受管版本。
- 清单会纳入 Git，不在其中保存密码、令牌或消息队列凭据。

## 只读检查

```bash
COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
"$COMPONENT_ROOT/scripts/product-config.sh" check
```

## 更新插件组件版本

```bash
(
  COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
  printf '请输入插件组件新版本号：'
  IFS= read -r PLUGIN_VERSION
  if [ -z "$PLUGIN_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$COMPONENT_ROOT/scripts/product-config.sh" set-version "$PLUGIN_VERSION"
)
```

## 更新 BifroMQ 兼容版本

```bash
(
  COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
  printf '请输入 BifroMQ 新兼容版本号：'
  IFS= read -r BIFROMQ_VERSION
  if [ -z "$BIFROMQ_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$COMPONENT_ROOT/scripts/product-config.sh" set-bifromq-version "$BIFROMQ_VERSION"
)
```

## 更新 Util 依赖版本

```bash
(
  COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
  printf '请输入 Util 依赖新版本号：'
  IFS= read -r UTIL_VERSION
  if [ -z "$UTIL_VERSION" ]; then

    exit 1
  fi
  "$COMPONENT_ROOT/scripts/product-config.sh" set-util-version "$UTIL_VERSION"

```

## 更新 Java 编译目标版本

```bash
(
  COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
  printf '请输入 Java 编译目标主版本：'
  IFS= read -r JAVA_MAJOR_VERSION
  if [ -z "$JAVA_MAJOR_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$COMPONENT_ROOT/scripts/product-config.sh" set-java-version "$JAVA_MAJOR_VERSION"
)
```

每个版本写入命令都会自动完成渲染和一致性检查，不需要紧接着再执行 `render`。BifroMQ 兼容版本和 Java 主版本还会执行脚本内置的兼容性约束校验。

## 手工修改清单后渲染

```bash
COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
"$COMPONENT_ROOT/scripts/product-config.sh" render
```

## 完整验证

```bash
COMPONENT_ROOT="$(git rev-parse --show-toplevel)/bifromq-plugin"
"$COMPONENT_ROOT/scripts/tests/product-config-test.sh"
"$COMPONENT_ROOT/scripts/product-config.sh" check
mvn --batch-mode -f "$COMPONENT_ROOT/pom.xml" -DskipTests clean package
git -C "$COMPONENT_ROOT" diff --check -- .
git -C "$COMPONENT_ROOT" status --short -- .
```

更新 Util 依赖版本前，先在 Util 仓库完成对应版本的本地安装。升级 BifroMQ 主版本时，先确认 SPI 和 Java 包兼容性，不能只修改清单版本号。

## 同步到其他发行仓库

同步代码时保留目标仓库自身的 `.thinglinks-product.env` 和清单中 `THINGLINKS_SYNC_PROTECTED_PATHS` 指定的文件。同步完成后，在目标仓库重新执行 `render`、`check`、配置回归和打包检查。

不要同步 `.git`、`target`、IDE 配置、依赖缓存和运行时临时文件。

## 写锁提示

`render` 和各版本更新命令属于写操作，避免并发执行。出现产品配置写锁提示时，先等待正在运行的命令结束，不手工删除仍由活动进程持有的锁引用。

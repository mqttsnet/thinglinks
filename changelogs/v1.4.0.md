# 1.4.0 (2026-07-22)

## Highlights

- Rebuilt device uplink processing around a staged message-bus pipeline, with Groovy pre-transform, online script debugging and version-aware thing-model resolution
- Added product-version publishing, device-bound version switching, named-device canary rollout, OTA orchestration and retry scheduling
- Added the northbound data-bridge rule engine, notification workflows, execution traces and operational dashboards
- Expanded video capabilities with GB28181 signaling, ISUP/JT1078 access, platform cascade, recording plans and SIP configuration
- Upgraded the admin console with device shadow, asset statistics, MQTT/WebSocket debugging, OTA management and large-screen visualization workflows

## Security

- Fixed authenticated remote code execution in the Groovy script registration/test flow by removing externally reachable test controllers and restricting dynamic execution to internal service calls ([GHSA-9vph-5ccx-qp9v](https://github.com/mqttsnet/thinglinks/security/advisories/GHSA-9vph-5ccx-qp9v), [CVE-2026-62853](https://github.com/advisories/GHSA-9vph-5ccx-qp9v))
- Fixed unauthenticated SQL injection in the TDengine TDS APIs by moving TDS operations behind internal routes, validating every dynamic identifier and binding values as parameters ([GHSA-gcj7-r5wv-348x](https://github.com/mqttsnet/thinglinks/security/advisories/GHSA-gcj7-r5wv-348x), [CVE-2026-62849](https://github.com/advisories/GHSA-gcj7-r5wv-348x))
- Updated the shared `thinglinks-parent` / `thinglinks-util` dependency line to `1.0.8.3`
- Replaced bundled credentials and map keys with explicit placeholders and added configuration checks to prevent secrets from entering production builds

## Fixed Issues

- Rule scripts now receive a per-device context, including `device.signKey`, so one product-level script can dynamically use each device's signing key without manual edits ([#105](https://github.com/mqttsnet/thinglinks/issues/105))
- Business dictionary queries now return only enabled items, preventing disabled instance addresses and other dictionary values from remaining selectable ([#109](https://github.com/mqttsnet/thinglinks/issues/109))
- Fixed tenant datasource connection leaks, production component registration, route re-registration, device-shadow WebSocket cleanup and product-model dialog layering
- Fixed notification resource registration, validation and multilingual copy across alarm and linkage workflows

### Added
- Uplink message-bus pipeline (`thinglinks-mqs-biz-bus`) with Groovy pre-transform and online script debugging
- Device downlink command channel (`DeviceDownlinkFacade`) and WebSocket session registry with downlink broadcast
- Northbound data-bridge rule engine (`thinglinks-rule-biz-bridge`): bridge rules, execution trace/steps, channel constants and dashboard summary
- Product version management (versions / publish records / change logs) and CA certificate audit log
- Video: GB28181 signaling event system, ISUP & JT1078 access, platform cascade, record plans and SIP config; protocol layer extracted into `thinglinks-video-biz-protocol`
- Admin console: OTA management, asset statistics, device shadow, product version pages and rule-script debug panel, with zh/en/ja locales
- BifroMQ plugins: event processors reorganized by package structure with 18 client-disconnect sub-types, HLC/UTC causal-clock semantics and per-client Kafka partition key
- Agent Skills install guide in README (per sub-project skills)

### Changed
- mqs rebuilt around the bus pipeline: legacy tcp/ws/mqtt/consumer handler suites retired; uplink handlers and device services relocated under `mqs.*` packages
- Gateway auth chain optimized with local resource caching (no per-request Redis, no O(N) matching); oauth login chain tuned
- Video device/channel management redesigned (`video_device` / `video_channel`)
- Web components relocated by application domain (`components/{iot,card,video}`); video page directories normalized
- Tenant init schema upgraded to the 91-table baseline (product versions, cert audit, bridge rules, video tables)
- Nacos templates: production-grade Druid pool (fail-fast, keep-alive, abandoned-connection guard), Kafka producer tuning with `max.block.ms` fast-fail, Seata tx-group key corrected; all credentials replaced with neutral placeholders
- Project version unified to 1.4.0
- Database delivery is now full-baseline only for 1.4.0; no generic incremental migration scripts are provided

### Fixed
- Tenant datasource initialization connection leak (database name now parsed from the JDBC URL instead of borrowing pooled connections)
- Production build: explicit AntD `Card`/`Typography` registration (tree-shaking degradation) and `radio-button` style-import mapping
- `IgnoreProperties` method-bucket traversal
- `ota_upgrades.sign_method` column added to tenant init scripts

### Removed
- Empowerment module (deprecated feature) across backend, schema and console
- Legacy RabbitMQ dead code and legacy WebSocket Observer/Subject suites
- Legacy product-event listeners superseded by the cache-evict event system

## Upgrade Notes

- New installations should initialize the default and tenant databases from the 1.4.0 full baselines
- Existing installations must prepare a database-specific migration plan and backup; do not apply the full baselines directly because they contain destructive table recreation statements
- Replace all credential, map-key and external-service placeholders before starting production services

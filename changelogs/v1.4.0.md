# 1.4.0 (2026-06-11)

### Added
- Uplink message-bus pipeline (`thinglinks-mqs-biz-bus`) with Groovy pre-transform and online script debugging
- Device downlink command channel (`DeviceDownlinkFacade`) and WebSocket session registry with downlink broadcast
- Northbound data-bridge rule engine (`thinglinks-rule-biz-bridge`): bridge rules, execution trace/steps, channel constants and dashboard summary
- Product version management (versions / publish records / change logs) and CA certificate audit log
- Video: GB28181 signaling event system, ISUP & JT1078 access, platform cascade, record plans and SIP config; protocol layer extracted into `thinglinks-video-biz-protocol`
- Admin console: OTA management, asset statistics, device shadow, product version pages and rule-script debug panel, with zh/en/ja locales
- BifroMQ plugins: event processors reorganized by package structure with 18 client-disconnect sub-types, HLC/UTC causal-clock semantics and per-client Kafka partition key
- Agent Skills install guide in README (per sub-project skills)

### Changed
- mqs rebuilt around the bus pipeline: legacy tcp/ws/mqtt/consumer handler suites retired; uplink handlers and device services relocated under `mqs.*` packages
- Gateway auth chain optimized with local resource caching (no per-request Redis, no O(N) matching); oauth login chain tuned
- Video device/channel management redesigned (`video_device` / `video_channel`)
- Web components relocated by application domain (`components/{iot,card,video}`); video page directories normalized
- Tenant init schema upgraded to the 91-table baseline (product versions, cert audit, bridge rules, video tables)
- Nacos templates: production-grade Druid pool (fail-fast, keep-alive, abandoned-connection guard), Kafka producer tuning with `max.block.ms` fast-fail, Seata tx-group key corrected; all credentials replaced with neutral placeholders
- Project version unified to 1.4.0

### Fixed
- Tenant datasource initialization connection leak (database name now parsed from the JDBC URL instead of borrowing pooled connections)
- Production build: explicit AntD `Card`/`Typography` registration (tree-shaking degradation) and `radio-button` style-import mapping
- `IgnoreProperties` method-bucket traversal
- `ota_upgrades.sign_method` column added to tenant init scripts

### Removed
- Empowerment module (deprecated feature) across backend, schema and console
- Legacy RabbitMQ dead code and legacy WebSocket Observer/Subject suites
- Legacy product-event listeners superseded by the cache-evict event system

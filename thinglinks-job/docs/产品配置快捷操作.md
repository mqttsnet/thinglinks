# 产品配置快捷操作

本文用于快速完成 Job 产品配置、组件版本、Util 依赖版本和 Java 编译版本维护。

文中的命令可以从 `thinglinks` 仓库内任意目录执行，包括 `thinglinks-job/docs` 目录。每个代码块都会通过 Git 自动定位单仓库根目录和 Job 组件根目录，可以直接复制执行。

## 配置入口

Job 组件根目录 `.thinglinks-product.env` 是产品身份、Job 组件版本、依赖版本、发行信息、授权信息和运行标识的配置来源。

| 维度 | 清单字段 | 用途 |
| --- | --- | --- |
| Job 组件版本 | `THINGLINKS_COMPONENT_VERSION` | Job 聚合工程自身的发布版本 |
| Util 依赖版本 | `THINGLINKS_UTIL_VERSION` | `thinglinks-parent` 与 XXL-Job Core 依赖版本 |
| Java 主版本 | `THINGLINKS_JAVA_VERSION` | Maven 编译目标版本 |
| 服务名称 | `THINGLINKS_SERVICE_NAME` | Spring 应用注册和服务发现名称 |

- 修改版本时使用对应的版本命令。
- 修改其他产品配置时只编辑 `.thinglinks-product.env`，随后执行 `render`。
- 不分别手工修改根 POM、Admin POM 或应用名称中的受管值。
- 清单会纳入 Git，不在其中保存密码、令牌、Nacos 地址或数据库凭据。

## 只读检查

```bash
MONOREPO_ROOT="$(git rev-parse --show-toplevel)"
JOB_ROOT="$MONOREPO_ROOT/thinglinks-job"
"$JOB_ROOT/scripts/product-config.sh" check
```

## 查看版本

```bash
MONOREPO_ROOT="$(git rev-parse --show-toplevel)"
JOB_ROOT="$MONOREPO_ROOT/thinglinks-job"
"$JOB_ROOT/scripts/product-config.sh" get-version
"$JOB_ROOT/scripts/product-config.sh" get-util-version
"$JOB_ROOT/scripts/product-config.sh" get-java-version
```

## 更新 Job 组件版本

```bash
(
  MONOREPO_ROOT="$(git rev-parse --show-toplevel)"
  JOB_ROOT="$MONOREPO_ROOT/thinglinks-job"
  printf '请输入 Job 组件新版本号：'
  IFS= read -r JOB_VERSION
  if [ -z "$JOB_VERSION" ]; then
    printf '版本号不能为空。\n' >&2
    exit 1
  fi
  "$JOB_ROOT/scripts/product-config.sh" set-version "$JOB_VERSION"
)
```


## 数据库初始化与升级

数据库初始化和增量脚本注意事项见 [`docs/db/mysql/README.md`](db/mysql/README.md)。新建数据库使用初始化脚本，已有数据库只能按顺序执行 `migration` 目录中的增量脚本。

```bash
MONOREPO_ROOT="$(git rev-parse --show-toplevel)"
JOB_ROOT="$MONOREPO_ROOT/thinglinks-job"
mysql -u root -p thinglinks_job < "$JOB_ROOT/docs/db/mysql/baseline/thinglinks_job.sql"
```

## 完整验证

```bash
MONOREPO_ROOT="$(git rev-parse --show-toplevel)"
JOB_ROOT="$MONOREPO_ROOT/thinglinks-job"
"$JOB_ROOT/scripts/tests/product-config-test.sh"
"$JOB_ROOT/scripts/product-config.sh" check
mvn --batch-mode -f "$JOB_ROOT/pom.xml" -DskipTests clean package
git -C "$MONOREPO_ROOT" diff --check -- thinglinks-job
git -C "$MONOREPO_ROOT" status --short -- thinglinks-job
```

更新 Util 依赖版本前，先在 Util 仓库完成对应版本的本地安装。

## 同步到其他发行仓库

同步代码时保留目标仓库自身的 `.thinglinks-product.env` 和清单中 `THINGLINKS_SYNC_PROTECTED_PATHS` 指定的文件。同步完成后重新执行 `render`、`check`、配置回归和 Maven 构建。

不要同步 `.git`、`target`、`.flattened-pom*.xml`、IDE 配置、依赖缓存和运行时临时文件。
